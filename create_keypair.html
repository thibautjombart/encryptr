Create keypair

Usage

create_keypair(path, password, quiet = FALSE, name = "id_encryptr")


change_password(path, password, quiet = FALSE)


read_private_key(path)


read_public_key(path)

Arguments

path
Path to the directory to store the key.
password
What do we do about passwords? Options are FALSE for no password, TRUE for prompting for a password, or a string value for a password (which will end up in things like your history so be careful).
quiet
Suppress printing of informative messages.
name
Leave this be for now.

Description

Create a pair of public and private keys, possibly with password protection. If you're trying to do the data workflow thing, use data_user_init instead, which wraps this but puts the key in the place the rest of the functions there will look.

Details

The functions read_public_key and read_private_key will read the public and private key stored at path. If the private key is password-protected then you will be prompted for a password.

The change_password function will change the password of an existing key. This works by encrypting the private key, so the actual key remains the same, only the password to lock it changes.
